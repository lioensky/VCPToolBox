
### 2) 运行端到端 smoke（真实 Host + 直连 stdio）

```powershell
node .\scripts\vcp_host_smoke_paperreader_rs.js "D:\谷歌下载\ai科研绘图.pdf"
```

## Closeout+（2026-04-05）

补充收口点：

- 递归 Global Map 支持 critic 二次校验产物：`reading/recursive_maps/global_map.critic.md` 与 `reading/recursive_maps/level_*/group_*.critic.md`
- 注意：通过 VCP Host 的 `configSchema` 注入 boolean 配置时，目前只识别 `true/false`（不识别 `1/0`）。例如启用 critic 应写：`PaperReaderRecursiveCritic=true`。
- 研究图执行已升级为 **DAG ready-queue 并发调度**，节点级并发上限由 `PaperReaderMaxConcurrentNodes` 控制（默认 3）。
- `plan_research` 生成的研究图已从“线性链”调整为真正的 DAG：`survey` 与 `read` 在 time-zero 可并行启动，后续节点通过 `depends_on`（DataDependency edges）表达真实依赖。
- `plan_research(scope=collection)` 已落地 **文档 fan-out 子图**：`read_i` / `retrieve_i` 扇出并发执行，随后由聚合 `read` / `retrieve` 节点进行 fan-in，保持 `compare` / `conflict_audit` / `merge` 的兼容输入契约。
- `plan_research(reading_mode=recursive)`（兼容 `mode=recursive`）已落地 **单文档递归子图化**：新增 `read_recursive` 节点，节点链路为 `survey/read -> read_recursive -> retrieve -> merge -> synthesize`。
- `read_recursive` 节点执行显式透传 `mode=recursive + resume_from_state=true`，把原本内嵌在 `read_document` 的递归 refine 阶段提升为 run 图可观测节点；节点产物显式包含 `global_map` / `reading_synthesis` / `final_report`。
- `node_input_refs` 的 `merge` 输入已按 `depends_on` 泛化推导，避免文档递归 DAG 与固定输入模板产生漂移。
- 修复 `reset_run -> resume_research_graph` 路径：`reset_run` 会重置 `graph.json` 的节点动态字段，`execute_run_dag` 只以 `graph_state` 判断 completed，避免 reset 后误读缺失的 `final_answer.json`（`os error 2`）。
- 新增回归测试覆盖 reset 后 resume（见 `paperreader-application` 的 end-to-end flow 测试）。
- 新增回归测试：`test_plan_and_run_recursive_document_subgraph`，覆盖 recursive 规划节点存在性、运行完成状态、`read_recursive` 节点结果和 merge 输入引用校验。

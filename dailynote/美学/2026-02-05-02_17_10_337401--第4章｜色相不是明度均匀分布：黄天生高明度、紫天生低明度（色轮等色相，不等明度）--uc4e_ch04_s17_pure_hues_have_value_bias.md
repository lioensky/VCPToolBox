# 第4章｜色相不是明度均匀分布：黄天生高明度、紫天生低明度（色轮等色相，不等明度）

## 美学本质

一句话：每个纯色都有“天生明度”——黄天然明亮、紫天然幽暗；色轮可以等间隔地排 hue，但饱和色并不会自动等间隔地排 value。

原文给出三件常被误判的事实：
- 明度常与“发光感（luminosity）”混在一起：反射多→看起来亮→高明度；吸收多→看起来暗→低明度；
- 六个光谱色本身就处在不同明度层级：黄最亮、紫最暗；
- 关键陷阱：饱和色在明度上并不等距（黄到绿的明度差远大于蓝到绿）。

它还指出一个常见“好心办坏事”的主张：有人想在黄↔蓝、黄↔红之间插更多“步进”来匹配明度差——但这会引发互补配对的几何问题；更根本的是：光谱的目的在于展示 hue 的全范围，插再多间隔也不会创造新 hue，把“步数不同”当成“hue 更多”是误解。

## 品味裁决

【俗套陷阱】把色轮当作等明度地图：以为相邻 hue 一定相近明暗。
为何廉价：你会在配色时遇到“关系对了但画面塌了”的问题——明度偏差在偷偷改写层级。

【优雅签名】先承认 hue 有明度偏置，再设计补偿：需要对齐明度时，就让某些 hue 以 tint 或 shade 形式出场。
精妙之处：你把“意外”变成“策略”：哪些颜色必须变浅/变深，变得可预期。

## 张力透视

【张力结构】等色相秩序（色轮） vs 不等明度现实（材料/视觉）。
【统一机制】分开两张地图：一张管 hue 顺序，一张管 value 层级；两者叠加才是可用的配色。
【平衡临界点】只看 hue 会塌结构；只看 value 会失去色彩性格。

## 跨媒介迁移

【灵魂（不变量）】系统里最危险的是“看似均匀但其实偏置”。
【皮相变体】
- 摄影：黄色区域容易“自带高光”，紫色容易“自带阴影”；
- UI：黄色按钮在同一背景下更显眼，紫色更显沉；
- 印刷：用白框隔离样本（原文提示的做法）能更客观地比较明度。
【边界条件】显示器与印刷对明度与色域的呈现不同，必须用目标媒介验证“补偿是否成立”。

## 审美唤醒

【审美模型】色轮回答“顺序”，明度回答“层级”。
【一句话启发】「别指望色相替你守住明暗——黄和紫天生就不在一个明度世界。」

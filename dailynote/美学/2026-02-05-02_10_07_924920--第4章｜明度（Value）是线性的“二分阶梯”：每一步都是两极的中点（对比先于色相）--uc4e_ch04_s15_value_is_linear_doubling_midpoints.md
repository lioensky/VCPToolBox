# 第4章｜明度（Value）是线性的“二分阶梯”：每一步都是两极的中点（对比先于色相）

## 美学本质

一句话：色相是圆的，但明度是线的——它从黑到白有起点与终点；更重要的是，明度对比不依赖色相存在：就算没有颜色，明暗也能让结构出现。

原文给出的“明度生成法”很值得记住：
- value=样本的相对明暗；hue 连续成环，而 value 线性递进；
- 白是最高 value，黑是最低 value，中灰是黑白中点；
- value scale 是一串“不断二分”的台阶：每一步都是两边的中点——比前一阶“暗一半”，比后一阶“暗两倍”。

## 品味裁决

【俗套陷阱】只盯 hue：以为换个颜色就能解决画面问题。
为何廉价：很多“看不清/没重点”的问题其实是 value 结构塌了，换 hue 只是换皮。

【优雅签名】先做 value 骨架：先把黑白灰的结构搭稳，再让 hue 上场。
精妙之处：你获得“可读性与焦点”的硬地基，颜色变成可控的情绪层。

## 张力透视

【张力结构】线性秩序（黑→白） vs 环形秩序（色相轮回）。
【统一机制】先线后环：明度负责“让东西出现”，色相负责“让东西有性格”。
【平衡临界点】
- 全靠明度：强但可能干；
- 全靠色相：艳但可能糊。

## 跨媒介迁移

【灵魂（不变量）】对比是信息结构的底层语法。
【皮相变体】
- 摄影：剪影、逆光、舞台光，本质都是 value 叙事；
- UI：可读性优先，靠 value 层级建立信息架构；
- 绘画：先做明度稿再上色，是对“对比先于色相”的承认。
【边界条件】当明度差过小，色相再漂亮也会“说不清”；反之明度差过大，会压扁色相的细腻。

## 审美唤醒

【审美模型】可读性 ≈ 明度对比 × 层级清晰度。
【一句话启发】「先让东西在明暗里出现，再让它在颜色里开口。」

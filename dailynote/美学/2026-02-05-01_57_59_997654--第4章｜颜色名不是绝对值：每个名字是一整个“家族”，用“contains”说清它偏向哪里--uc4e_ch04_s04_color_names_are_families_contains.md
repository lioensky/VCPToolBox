# 第4章｜颜色名不是绝对值：每个名字是一整个“家族”，用“contains”说清它偏向哪里

## 美学本质

一句话：一个颜色之所以叫“黄”，只是因为黄在它里面占主导——但几乎所有样本都混着别的 hue；比较一出现，隐藏的偏色就暴露出来。

原文给出一种非常可用的判断语言：
- 颜色名代表的是“相关 hues 的家族”，不是一个点；
- 同类相邻对比，会让你突然看到“这个黄里有点绿、那个黄里有点橙”；
- 用 “contains” 来说颜色（This yellow contains some orange）能同时指出主 hue 与次 hue。

## 品味裁决

【俗套陷阱】把颜色命名当真理：执着于“它到底是不是黄”，用标签压过观察。
为何廉价：你把连续谱硬切成离散格子，于是失去对“偏向”的敏感度。

【优雅签名】把命名当索引：先承认它属于某个家族，再用“contains”说出它被谁轻轻改写。
精妙之处：你不再用“更黄/更绿”的模糊形容，而是能给出结构化描述——主 hue + 次 hue。

## 张力透视

【张力结构】类别（黄/绿） vs 连续（从黄绿到绿黄的无穷过渡）。
【统一机制】用“家族 + 偏向”兼容两者：名字提供类别归属，contains 提供连续方向。
【平衡临界点】
- 只谈类别：看不见细微差别；
- 只谈连续：沟通成本爆炸，无法快速共识。

## 跨媒介迁移

【灵魂（不变量）】颜色判断是相对的：没有对比就没有偏色。
【皮相变体】
- 摄影：白平衡/肤色校正常用“偏绿/偏橙”，本质就是在说 contains；
- 绘画/印刷：配色与调色靠“往里加一点蓝/加一点红”来塑形，先得看见次 hue；
- UI/品牌：一套色阶如果全是“同一个蓝”，那只是名字相同；真正一致性是偏向一致。
【边界条件】当环境光/材质导致偏色变化时，contains 必须在“指定观察条件”下讨论，否则描述会漂移。

## 审美唤醒

【审美模型】颜色命名 = 主导 hue（归属） + 次要 hue（偏向） + 属性（明度/饱和度）。
【一句话启发】「别问它是不是黄——问它的黄里‘含’了什么。」

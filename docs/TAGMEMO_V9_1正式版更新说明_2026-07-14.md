# TagMemo V9.1 正式版更新说明：从经验有效到工程收敛

> 发布日期：2026-07-14  
> 发布状态：正式版 / 全量生产版本  
> 对外版本标识：`v9`  
> 内部算法版本：`v9.1`  
> 历史生产基线：V8.3  
> 当前生产结论：V9.1 已全量覆盖 V8.3，V8.3 不再作为活动运行轨道

---

## 1. 正式发布结论

经过多轮固定测试集复跑、独立 Agent 测试、工程补丁消融和真实记忆召回主观评测，TagMemo V9.1 已完成从 Beta 实验版本到单轨生产版本的收敛。

本次正式发布确认以下结论：

1. V8.3 的总体方向是正确的，浪潮传播、叙事方向、概念锚、虫洞和势能场确实能够提供 KNN 之外的结构召回能力；
2. V8.3 的主要问题集中在工程落地、状态一致性、传播噪声控制和派生资产管理，而不是基础方向错误；
3. 将 V9.1 的三项工程修复补丁下放到 V8.3、但不移植 V9.1 数学算法后，V8.3 与 V9.1 的客观精度差距缩小到约 0.2%；
4. 在真实召回的主观评分中，上述修复版 V8.3 与 V9.1 的差距小于 5%；
5. 相比最古老的 V8.3 实现，V9.1 中央候选区提升约 30%，边界候选区提升尤其明显，部分场景达到数倍；
6. V9.1 的收益不是单一数学公式带来的神奇跃迁，而是正确方向、工程修复、传播收敛和资产一致性共同作用的结果；
7. V9.1 已具备替代 V8.3 的充分依据，现正式成为唯一生产轨道。

这次升级不是对 V8.3 的否定，而是对 V8.3 技术路线的工程收敛。

## 2. 验证规模

正式结论来自多轮、重复和不同执行主体的测试，而不是单次演示结果。

已完成的主要验证包括：

- 350 组固定单元测试集，完整复跑 4 次；
- 由 Nova 独立执行 50 组测试，完整复跑 2 次；
- 对同一批查询进行 V8.3、修复版 V8.3 与 V9.1 的对照；
- 对中央候选、边界候选、长尾概念和跨域关联分别观察；
- 对客观排名指标和真实记忆召回的主观有效性分别评分；
- 首次加入 Zero-shot 原创概念私有知识场景；
- 对并发查询、派生训练、文件写入和 SQLite 协调进行生产环境验证。

按完整测试轮次计，本轮共覆盖 1,500 组次执行。其中固定 350 组测试承担版本间稳定对照，Nova 独立测试用于降低单一测试设计者造成的偏差。

这些数据足以支持生产切换，但不意味着所有知识域、embedding 模型和参数组合都已获得统计完备证明。本文中的百分比均限定于本轮测试环境和评测口径。

## 3. 最关键的消融实验：把修复下放给 V8.3

为了判断 V9.1 的收益究竟来自新数学算法，还是来自工程实现质量，本轮进行了关键消融：

> 将 V9.1 的三项函数级修复补丁逐一下放到 V8.3，但保留 V8.3 原有数学传播算法。

结果是明确的：

- 每项修复都表现出正收益；
- 修复后的 V8.3 显著优于最古老的 V8.3；
- 修复版 V8.3 与完整 V9.1 的客观精度差距约为 0.2%；
- 在真实召回主观评分中，两者差距小于 5%。

这组结果具有两层意义。

第一，V8.3 的算法方向没有根本错误。它已经捕获了真实存在的结构信号，历史效果并非偶然。

第二，V8.3 曾经存在显著的工程折损。请求级状态污染、派生缓存语义、传播噪声和并发写入等问题，会掩盖正确算法本应产生的收益。

因此，V9.1 最准确的定位不是“推翻 V8.3 的全新理论”，而是：

> 在保留 V8.3 正确机制的基础上，修复工程折损，并将传播动力学收敛到更稳定、更可控的生产形态。

V9.1 数学变化仍有独立价值，尤其体现在边界候选、枢纽污染、立即回流和有限时域稳定性上；但消融结果要求我们避免把全部收益错误归因于单一公式。

## 4. Zero-shot 原创概念私有场景

本轮首次引入针对原创概念和私有知识体系的 Zero-shot 单元测试。

### 4.1 测试定义

该场景刻意使用：

- 完全自造的词汇；
- 完全自造的概念关系；
- 不存在于公共语料中的私有知识体系；
- 对模型预训练知识而言近似完全零命中；
- 无法依赖常见实体、百科知识或词面先验完成猜测的查询。

这一测试与普通知识库问答不同。普通查询可能让 embedding 模型凭借预训练语义直接给出很强的近邻；原创概念场景则尽可能剥离这类先验，迫使检索系统依赖知识库内部已经形成的关系结构。

### 4.2 实测结果

在本轮 Zero-shot 原创概念测试中，TagMemo 浪潮记忆对纯 KNN 和常规 rerank 表现出近乎压倒性的优势。

主要观测包括：

- KNN 在词面和预训练语义均缺乏支点时容易失去稳定方向；
- 常规 rerank 只能重排已经进入候选池的内容，无法充分补回未被近邻召回的结构关联；
- TagMemo 能沿私有知识体系中的共现方向、叙事顺序和局部传播路径找到相关记忆；
- 优势不仅出现在尾部补充，也出现在决定答案上下文的关键前置记忆上；
- 即使原创术语本身没有公共世界知识支撑，知识库内部建立的关系仍可驱动有效寻址。

这证明 TagMemo 并非仅仅把 embedding 相似度换一种方式重新排序。它确实利用了向量近邻之外的结构信息，并在缺乏预训练先验时展现出独立召回能力。

### 4.3 “因 → 果”排序倾向

测试中还观察到一个反复出现的宏观行为：

> TagMemo 倾向于按照“因 → 果”的线性叙事方向组织候选。

这会带来一种与传统名词解释评测不同的排序偏好。

对于“某概念是什么”一类表面查询，KNN 往往优先返回直接定义该名词的文本；TagMemo 则可能先返回导致该概念出现、解释其形成条件或构成其前置背景的记忆。

如果只用“名词定义是否排在第一位”评分，这种行为可能被误判为名词解释排位下降。但在连续记忆和 Agent 认知场景中，前因往往比孤立定义更重要，因为它决定：

- 当前状态为何形成；
- 后续事件应如何解释；
- 哪些历史约束仍然有效；
- Agent 应沿哪条叙事链继续推理。

因此，TagMemo 的排序目标更接近“恢复可继续推理的记忆链”，而不只是“返回词面最像问题的文本块”。

### 4.4 可以得出的结论与边界

Zero-shot 结果强力支持以下结论：

1. 浪潮引擎捕获了纯向量距离无法充分表达的内部关系信号；
2. 有向共现和有限传播可以拟合一部分线性叙事与逻辑联想行为；
3. “因 → 果”倾向具有跨原创概念样本重复出现的宏观稳定性；
4. TagMemo 的收益在预训练知识失效时反而更加清晰；
5. 浪潮动力学不是装饰性比喻，它对应了可测试、可消融、可重复观察的实际计算行为。

但这些结果仍不能直接证明：

- 人类逻辑联想在微观数学上等价于流体力学；
- 当前传播方程是认知过程的唯一或完整模型；
- 任意数据库和任意参数下都必然获得相同优势；
- 宏观有效性已经自动构成微观机制的严格证明。

更严谨的表述是：

> 目前尚无法从微观数学层面完整解释全部涌现行为，但在受控 Zero-shot 私有知识环境中，浪潮动力学表现出可重复的宏观可靠性，并且这种可靠性不能由纯 KNN 或候选内 rerank 充分解释。

微观解释尚未闭合，不削弱已经获得的宏观工程证据；但宏观工程证据也不应被夸大为完成了认知科学层面的最终证明。

## 5. V9.1 传播动力学

V9.1 保留 V8.3 已验证有效的图事实和方向结构，同时对传播过程进行三项收敛。

### 5.1 固定出流预算与预算内虫洞

累计共现证据先进行对数压缩：

\[
C_{ij}=\log(1+\lambda W_{ij})
\]

随后所有出边在同一节点的固定预算内竞争：

\[
P_{ij}=m_{\mathrm{out}}\frac{g_{ij}}{\sum_k g_{ik}}
\]

并满足：

\[
\sum_j P_{ij}\leq m_{\mathrm{out}}\leq1
\]

虫洞可以提高预算内优先级，但不能在归一化后创造额外传播质量。

### 5.2 入流枢纽校正

固定出流解决了单个节点释放过多质量的问题，但通用标签仍可能从大量来源吸积质量。

V9.1 统计目标节点的全图入流，并以正入流中位数为基准实施夹逼幂律校正。该校正只改变同一行内的相对选择，不突破节点总预算。

其主要作用是：

- 抑制高频通用标签形成引力井；
- 保护低频但具有明确局部支持的概念；
- 降低边界候选被枢纽污染的概率；
- 保留真实核心概念，不执行无限惩罚。

### 5.3 软非回溯

传播状态由单纯节点状态升级为携带前驱信息的边状态。

对立即回流：

\[
u\rightarrow i\rightarrow u
\]

施加默认 0.15 的质量系数。普通汇聚、跨节点推进和非立即回流不受该规则影响。

这不是禁止回忆旧节点，而是抑制二节点之间的短周期自激振荡。

### 5.4 归一化有限时域场

V9.1 不再无权累计每一跳能量，而使用归一化几何有限脉冲响应：

\[
w_t=\frac{\gamma^t}{\sum_{r=0}^{T}\gamma^r}
\]

\[
E=\sum_{t=0}^{T}w_th^{(t)}
\]

默认 \(\gamma=0.60\)。

这使近距离叙事证据保持主导，同时避免远跳残余能量仅凭累计次数污染候选尾部。

## 6. 稳定残差与单轨派生资产

V9.1 将内生残差定义为单位化标签向量相对于局部邻域基底的不可解释比例，并保存稳定原始测量：

\[
0\leq r_i\leq1
\]

锚增益使用固定单调映射：

\[
g(r)=\operatorname{clip}(g_0+kr^\gamma,g_{\min},g_{\max})
\]

该设计不再依赖全库 Min-Max、median 或 MAD 对全部节点进行重新标定。因此，新增离群标签不会改变所有历史标签的物理量尺度。

正式版采用单轨资产策略：

- V9.1 是唯一可发布的活动 ArtifactBundle；
- residual、传播核、pairwise view 和实际配置作为整体原子发布；
- 每个请求固定持有一次解析得到的不可变资产包；
- 后台重建不会让进行中的请求读取新旧混合代际；
- 显式请求退休版本会返回 `TAGMEMO_VERSION_RETIRED`；
- V8.3 物理兼容列仅用于旧数据库安全迁移，不再产生或读取 V8.3 运行值；
- 全量训练完成后清理退休派生状态和兼容数值。

## 7. SQLite、Rust 与统一写入协调

V9.1 正式版不仅更新召回算法，也完成了知识库写入路径的统一协调。

过去多个 Agent 同时工作时，可能出现以下竞争：

- Agent A 正在查询或恢复向量索引；
- Agent B 正在写入 DailyNote；
- watcher 正在将文件变化写入 SQLite；
- Rust 正在生成 pairwise similarity 或 intrinsic residual；
- JS 与 Rust 分别持有 SQLite/WAL 连接并进行 checkpoint。

这些操作如果缺乏统一时序，可能导致持库线程竞争、瞬态 malformed 视图、重复入库或进程崩溃。

现在 KnowledgeBaseManager 提供统一的 DailyNote/外部文件 mutation 协调入口，其保证：

1. 外部文件修改按 FIFO 串行；
2. 写文件前等待 JS 批处理、删除任务、Rust 写租约和索引恢复进入安全状态；
3. 文件提交后主动进入索引队列；
4. 可等待 SQLite 与 Vexus 索引完成更新后再结束事务；
5. 删除、移动和重命名按“先删除旧路径，再提交新路径”表达最终文件真相；
6. 前台可选择在文件提交后立即返回，而后台入库仍保持有序完成；
7. Rust 派生写必须通过统一租约进入数据库；
8. Rust 写后由 JS 统一执行健康检查与 WAL checkpoint；
9. 对可恢复的 WAL/SHM 瞬态异常执行二阶段重开连接复检；
10. 真正损坏时停止 watcher 和批处理，避免重试风暴扩大损坏。

这使“查询记忆”和“写入记忆”不再是彼此独立、可能争抢持库线程的两套流程，而是同一数据库协调器下的有序操作。

## 8. 生产行为变化

正式版的生产行为如下：

- 默认且唯一 TagMemo 版本为 V9.1；
- 对外请求仍使用 `v9`，无需增加新的协议版本名；
- `tag_boost` 后缀 `+` 继续表示启用 Potential Field 重排；
- 显式旧版本请求不再静默回退；
- 请求级增强向量与 energy field 可在多个日记本查询中复用；
- 多日记本搜索先形成虚拟联合索引，再执行统一全局 Top-K；
- Potential Field 使用查询级 energy field，禁止依赖可能被并发请求覆盖的全局最近状态；
- 派生资产按模型签名、图代际、算法版本和实际配置签名；
- 新增唯一标签达到约 1% 阈值后，经过静默期触发增量训练；
- 管理端可主动请求完整自学习训练；
- 启动阶段优先恢复服务，重型派生刷新在冷却窗口后执行；
- 热缓存完整且无变更时跳过无意义的重复训练。

## 9. 如何理解 V9.1 的收益

本轮证据不支持“V9.1 仅靠新数学算法提升一切”，也不支持“V9.1 只是修 Bug，数学变化没有意义”。

更符合实验结果的分解是：

\[
\text{V9.1 收益}
=
\text{V8.3 正确方向}
+
\text{工程修复}
+
\text{资产一致性}
+
\text{传播噪声收敛}
+
\text{有限预算动力学}
\]

其中：

- 中央候选提升主要受工程正确性和状态一致性影响；
- 边界候选提升更能体现枢纽校正、软非回溯和有限时域场的价值；
- Zero-shot 原创概念优势证明图内部结构能够在预训练先验缺席时独立工作；
- “因 → 果”排序倾向说明系统优化目标更接近连续记忆链，而不是孤立名词解释；
- V8.3 修复版与 V9.1 的小差距证明旧路线有效；
- V9.1 更稳定的边界表现和单轨工程能力构成最终替换 V8.3 的理由。

## 10. 已知边界与后续研究

V9.1 已满足生产发布标准，但仍存在值得继续研究的问题：

1. “因 → 果”偏好对名词解释型查询可能造成直接定义排位下降；
2. 不同知识任务可能需要区分“定义优先”和“记忆前因优先”的读出策略；
3. Zero-shot 原创概念测试应继续扩大样本数和知识体系类型；
4. 需要进一步分离传播核收益与 Potential Field 二次读出收益；
5. 需要记录不同 embedding 模型下的重复性；
6. 宏观可靠性尚缺少完整的微观动力学证明；
7. 主观评分仍应继续由多执行主体、盲测和固定标注规范约束；
8. 边状态上限触发频率和 P50/P95 延迟仍需长期监控。

这些问题属于正式版之后的研究与调优方向，不再构成 V9.1 替代 V8.3 的阻塞项。

## 11. 升级与兼容说明

升级到 V9.1 时：

- 不需要重新生成全部 chunk embedding；
- 不需要复制知识库；
- 现有 SQLite 会通过附加式 schema migration 增加缺失字段；
- 旧 V8.3 物理列可保留，但不再参与生产计算；
- 首次完整训练会生成 V9.1 residual 和 pairwise artifact；
- 训练期间 Rust 写任务必须获取 KnowledgeBaseManager 租约；
- 旧版本调用方应移除显式 `v8_3` 请求；
- 若 native Rust 二进制较旧，需要重新构建 `rust-vexus-lite`。

建议升级后执行一次主动完整训练，以清理退休资产并确认当前模型签名下的派生缓存完整。

## 12. 最终定义

TagMemo V9.1 是浪潮记忆从双轨实验进入单轨生产的正式版本。

它验证了三件事：

1. V8.3 的方向正确，其历史收益不是偶然；
2. 正确算法如果缺乏工程一致性，会损失大量本应存在的效果；
3. 当向量模型面对完全原创、公共语料零命中的私有知识体系时，有向图传播可以提供纯 KNN 和常规 rerank 无法替代的记忆寻址能力。

因此，V9.1 的正式结论不是“已经从微观数学上解释了认知”，而是：

> 浪潮动力学已经在重复测试、工程消融、真实召回和 Zero-shot 私有知识场景中表现出稳定、可测量、可复现的宏观有效性。

从本版本开始，V9.1 全量覆盖 V8.3。V8.3 完成了作为机制发现者和生产基线的历史任务；V9.1 则承接这些正确机制，并以更可靠的工程底座、更稳定的传播边界和更明确的资产语义继续演进。
